Solaris Installation Warnings

The pkgadd command can generate various warning messages during the installation. Several warning messages are to be expected. The first one will be: 

The following files are already installed on the system and are being
used by another package:

        /opt (attribute change only)
        /opt/ishmail (attribute change only)
        /usr (attribute change only)
        /usr/bin (attribute change only)
        /usr/share (attribute change only)
        /usr/share/man (attribute change only)
        /usr/share/man/man1 (attribute change only)

Do you want to install these conflicting files [y,n,?,q] n

To be safe, the package is explicitly specifying that common directories should be installed. Since these (most likely) already exist, you'll see the warning. It doesn't matter if you install these files.

The next warning will be: 

The following files are being installed with setuid and/or setgid
permissions or are overwriting files which are currently
setuid/setgid:
        /opt/ishmail/1.0.5/bin/ishmail.real 

Do you want to install these setuid/setgid files [y,n,?,q] y

The Ishmail program needs setgid privilege in the mail group, so that it can coordinate update of mail folders with the system's mail transport subsystem. You should answer y to this question.

The next warning will be: 

This package contains scripts which will be executed with super-user
permission during the process of installing this package.

Do you want to continue with the installation of this package [y,n,?] y

The package contains two scripts, called request and postinstall, which do some housekeeping during the installation. The pkgadd command will run them with root privilege. If you are suspicious about what these scripts will do, use the pkgtrans command to extract the contents of the package without actually installing it (and running the privileged scripts). Then, look in the /var/spool/pkg/ishmail/install directory for the request and postinstall scripts. You may examine them to assure yourself that they will not abuse their root privilege.
